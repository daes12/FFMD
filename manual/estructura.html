Estructura

Principio

FFMD nació de la necesidad de utilizar un Framework para PHP que fuera rápido, ligero y utilizara pocos recursos del servidor. Después de utilizar unos pocos (y muy buenos) inicié la tarea de crear un Framework personalizado el cual inluyera solamente las clases base y librerias necesarias. De ello surgió FFMD y lo hago público para todo aquel que desee utilizar un Framework libre de decenas (cientos quizás) de plugins, librerias y objetos que nunca utilizará y al contrario utilizan espacio y recursos.

FFMD ha sido diseñado para utilizar el modelo MVC (separando la lógica de la presentación) y la POO.
El núcleo de FFMD es tan pequeño que requiere solamente de unos pocos archivos de configuración y sistema.

Directorios base

FFMD posee la siguiente estructura de archivos: 

Aplicación (Área reservada para alojar los archivos de nuestra aplicación)
-->Controlador
-->Modelo
-->Vista

Configuración (Archivos contenedores de las configuraciones aplicables al sistema)

Recursos (Todos los elementos que necesite tu aplicación)
-->css
-->img
-->js

Sistema (Núcleo del sistema)

Directorio Aplicación

Espacio reservado para los archivos de nuestra aplicación web.
Es posible (y recomendable) crear subdirectorios para organizar mejor los archivos; p.ej.: 

Aplicación
-->Controlador
-->-->Multimedia
-->-->Libros
-->Modelo
-->Vista
