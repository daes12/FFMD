FFMD Manual de Usuario v. 0.04

URL

Este Framework está diseñado para trabajar con URL amigables del tipo: sitio.com/usuario/daniel; en la cual se encuentra la página solicitada (usuario) y la variable que debe procesar (daniel).

.htaccess

En la carpeta raíz creamos un archivo .htaccess el cual se encargará de enviar todas las solicitudes a nuestro archivo index.php. La estructura propuesta para este archivo es la siguiente: 

RewriteEngine On
RewriteBase /

#Envio de todas las solicitudes a index.php
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ /index.php?variables=$1 [L]

IndexIgnore *

Controlador y Variables

El Framework dividirá la información de la URL para identificar el Controlador solicitado y las Variables.

Si el Controlador se encuentra en una subcarpeta debe indicarse en la URL; p.ej.:
Si el Controlador "buscar" se encuentra en la carpeta "musica" la URL será así: 

sitio.com/musica/buscar

Si se desea enviar una o más variables se agregan en la URL después de la ruta hacia el Controlador; p.ej.:
Si al ejemplo anterior le deseamos enviar las variables "nuevo" y "pop" la URL será así: 

sitio.com/musica/buscar/nuevo/pop

Las variables serán enviadas al Controlador en forma de array. Tomando como base el ejemplo anterior, las variables tomaran la siguiente estructura: 
array(
  0 => "nuevo",
  1 => "pop"
)

Caracteres Permitidos

Los únicos caracteres permitidos en la URL por el Framework: Caracteres alfanuméricos y el guión bajo (A-Za-z0-9_).
El Framework bloqueará las URLs que contengan caracteres no permitidos.
